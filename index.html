    FAQ Accordion

    Toggling visibility with classList.toggle() and CSS transitions.

    The DOM (Document Object Model) is the browser's live, in-memory representation of your HTML page as a tree of objects (nodes). JavaScript uses document.querySelector() to select nodes, then reads or writes their properties to make the page react.

    const is block-scoped and signals that the binding will never be reassigned — perfect for DOM references. var is function-scoped and prone to hoisting bugs. Default to const, use let only when the value must mutate (e.g. a counter), and avoid var entirely.

    Every interactive element follows Input → Process → Output. A user click is the input, your JavaScript function is the process (it reads state and applies logic), and the DOM mutation is the output (a visible change). Understanding IPO is understanding all of UI engineering.

    Always prefer textContent when inserting plain text — it safely escapes all HTML and protects against Cross-Site Scripting (XSS) attacks. Only use innerHTML when you genuinely need to inject markup, and even then, sanitize the input.

    Decoupling separates JavaScript logic from CSS presentation. Use js- prefixed classes as JavaScript hooks (never style them), and is- prefixed classes to define visual states (e.g. is-active, is-hidden). This makes your code predictable and maintainable.
